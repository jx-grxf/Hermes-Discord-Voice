# OpenClaw Discord Voice Bridge v1.0.4

## Highlights

- Refactored the Discord voice bridge into focused modules for embeds, handlers, helpers, listen turns, verbose streaming, and TTS configuration.
- Added Auto-listen Beta mode so a prepared voice session can listen while the bot is idle.
- Added Piper TTS as a first-class selectable provider alongside macOS `say` and ElevenLabs.
- Updated the runtime to Node.js 22+ and fixed macOS native addon startup when commands are launched from Codex.
- Renamed the project and docs to OpenClaw-Discord-Voice.

## Fixed

- Fixed the `@snazzah/davey` native binding startup failure on macOS by routing project scripts through a normal external Node.js runtime when the Codex app's embedded signed Node process is detected. The failure happened because macOS rejected loading the ad-hoc signed native addon into the hardened Codex Node process due to mismatched code-signing identity.
- Fixed public-facing `formatAge` output so release/docs-visible status strings are English instead of German.
- Fixed stale voice session reuse after reconnects or channel changes by cleaning up old OpenClaw sessions before creating a replacement.
- Fixed overly broad diagnostic path handling by centralizing Piper binary and model path resolution.
- Fixed info/status fallback text so the requesting Discord user is reflected when no voice session exists.

## Improved

- Improved maintainability by splitting the large Discord handler surface into smaller modules with focused tests.
- Improved Discord status embeds, join controls, TTS provider switching, and verbose-mode presentation.
- Improved startup and doctor coverage for environment variables, local binaries, Whisper models, and Piper assets.
- Improved CI by moving the project workflow to Node.js 22.
- Improved repository hygiene with `.codexignore`, `.node-version`, stricter runtime metadata, and updated dependency locks.
